将user_id和session_id作为上下文信息传递给智能体，让智能体能够识别当前用户和会话，提供更个性化的服务。

### 2. **历史上下文增强**

#### 更新的 `_build_context_from_history()` 方法
```python
def _build_context_from_history(self, history: list, current_message: str, user_id: str = None, session_id: str = None) -> str:
    """从历史对话构建上下文，包含用户信息"""
    
    # 构建用户信息上下文
    user_info = f"""当前用户信息：
- 用户ID: {user_id or '未知'}
- 会话ID: {session_id or '未知'}
- 当前时间: {time.strftime('%Y-%m-%d %H:%M:%S')}

"""
    
    # 构建完整上下文提示
    context_prompt = f"""{user_info}以下是我们的对话历史，请基于上下文理解我的当前需求并给出回应：

{full_context}

请基于以上对话历史和当前用户信息，理解我的当前需求并提供合适的回应。"""
    
    return context_prompt
```

## 🔄 上下文结构

### 完整的上下文示例
```
当前用户信息：
- 用户ID: user123
- 会话ID: session456
- 当前时间: 2024-03-26 18:30:00

以下是我们的对话历史，请基于上下文理解我的当前需求并给出回应：

用户: 你好
助手: 您好！我是智能助手，有什么可以帮助您的吗？
用户: 我要执行任务
助手: 好的，请告诉我您要执行什么任务？

请基于以上对话历史和当前用户信息，理解我的当前需求并提供合适的回应。
```

### 上下文层次结构
```
1. 用户身份层
   - 用户ID: 唯一标识
   - 会话ID: 会话标识
   - 当前时间: 时间上下文

2. 对话历史层
   - 历史消息: 之前对话内容
   - 角色标识: 用户/助手区分
   - 时间顺序: 对话时序

3. 当前消息层
   - 用户输入: 当前需求
   - 图片信息: 可选的视觉内容
   - 上下文提示: 明确的处理指导
```

## 🏗️ 技术实现

### 1. **上下文构建流程**
```
1. 获取用户信息 (user_id, session_id)
   ↓
2. 构建用户上下文 (ID + 时间)
   ↓
3. 获取对话历史 (短期记忆)
   ↓
4. 构建历史上下文 (历史对话)
   ↓
5. 合并上下文 (用户 + 历史 + 当前消息)
   ↓
6. 传递给智能体处理
```

### 2. **数据传递链路**
```
API Request → ChatService → MainDeepAgent → DeepAgents
     ↓              ↓              ↓              ↓
  user_id      获取历史        构建上下文      处理消息
  session_id    传递参数        包含用户信息    返回响应
```

### 3. **错误处理机制**
```python
# 用户信息缺失处理
user_id or '未知'
session_id or '未知'

# 上下文构建失败处理
return f"""当前用户信息：
- 用户ID: {user_id or '未知'}
- 会话ID: {session_id or '未知'}
- 当前时间: {time.strftime('%Y-%m-%d %H:%M:%S')}

用户消息: {current_message}"""
```
