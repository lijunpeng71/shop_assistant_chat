# 📋 任务确认功能说明

## 🎯 功能概述

为任务智能体添加了任务确认机制，在执行任何任务之前，系统会先向用户说明任务内容，询问用户是否确认执行，只有在用户明确同意后才开始执行任务。

### 2. 主智能体处理逻辑

#### 任务识别和确认
```python
async def process_message(self, message: str, user_id: str = None, session_id: str = None, image_url: str = None):
    message_lower = message.lower()
    
    # 检查任务相关关键词
    if any(keyword in message_lower for keyword in ["任务", "执行", "检查", "陈列", "库存", "盘点"]):
        # 使用任务确认机制
        return self._handle_task_confirmation(message, user_id, session_id, image_url)
```

#### 任务确认处理
```python
def _handle_task_confirmation(self, message: str, user_id: str, session_id: str, image_url: str = None):
    # 分析用户意图，识别具体任务
    if any(keyword in message_lower for keyword in ["库存", "盘点"]):
        task_info = {
            "task_type": "inventory_check",
            "task_name": "库存盘点任务",
            "description": "对当前库存进行全面盘点...",
            "estimated_time": "约15-20分钟",
            "requirements": ["需要访问库存记录", "需要实地检查商品"]
        }
    
    return {
        "type": "task",
        "message": "📋 **任务确认**\n\n根据您的需求，我识别到将要执行以下任务...",
        "data": {
            "requires_confirmation": True,
            "task_info": task_info,
            "confirmation_options": ["确认执行", "同意执行", "取消", "放弃任务"]
        }
    }
```

## 📊 支持的任务类型

### 1. 库存盘点任务
- **触发关键词**: "库存", "盘点"
- **任务描述**: 对当前库存进行全面盘点，包括商品数量、保质期检查、陈列位置确认等
- **预计耗时**: 约15-20分钟
- **执行要求**: 需要访问库存记录、需要实地检查商品、需要记录盘点数据

### 2. 销售数据分析任务
- **触发关键词**: "销售", "数据", "分析"
- **任务描述**: 分析近期的销售数据，包括销售趋势、热门商品、销售时段分析等
- **预计耗时**: 约10-15分钟
- **执行要求**: 需要访问销售记录、需要数据分析工具、需要生成报告

### 3. 综合任务执行
- **触发关键词**: "任务", "执行"
- **任务描述**: 根据您的具体需求执行相应任务，包括任务规划、执行和结果汇报
- **预计耗时**: 根据任务内容确定
- **执行要求**: 需要明确任务内容、需要执行权限、需要结果反馈

### 4. 冰柜陈列检查（特殊处理）
- **触发关键词**: "检查冰柜", "冰柜陈列", "检查陈列", "冰柜检查"
- **特殊要求**: 需要提供图片URL
- **处理方式**: 独立的图片处理流程

## 🔧 用户交互

### 确认选项
用户可以通过以下方式响应：

#### 确认执行
- "确认执行"
- "同意执行"
- "好的，执行"
- "开始执行"

#### 取消任务
- "取消"
- "放弃任务"
- "不执行"
- "停止"

#### 询问详情
- "详细说明"
- "具体内容"
- "需要什么"
- "怎么做"

### 响应处理

#### 用户确认后
系统会：
1. 开始执行任务
2. 提供详细的执行结果
3. 给出改进建议
4. 建议适当的奖励

#### 用户取消后
系统会：
1. 停止任务执行
2. 确认取消操作
3. 提供其他选择建议
